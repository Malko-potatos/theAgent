# Architecture Decision Records

Use this folder for decisions that should remain auditable over time.

Recommended filename:

```text
NNNN-<decision-slug>.md
```

Each ADR should include:

- status: proposed, accepted, superseded, or rejected
- context
- decision
- alternatives considered
- consequences
- evidence or related run notes

Prefer adding a new ADR when a decision changes instead of silently rewriting prior history.
